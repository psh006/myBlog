# 搭建 Vue项目

# 搭建前准备

# 查看node.js环境

C:\Users\Admin>node -v
v12.16.1

如果没有node.js环境,安装node.js,参考这里

# 查看vue脚手架版本

C:\Users\Admin>vue -V
@vue/cli 4.2.3

如果vue脚手架低于3.0.0,应该删除当前版本,安装最新版本

npm uninstall vue-cli -g
npm install -g @vue/cli

如果没有安装过vue脚手架,直接安装最新版本

npm install -g @vue/cli

# 搭建项目

cmd 进入想要搭建项目的目录下,执行以下命令

D:\psh-workspace>vue create myvuedemo

出现以下选择,按上下键进行选择,defaults,是默认配置,此时选择第二选项(Manually...),选择完成后,回车

? Please pick a preset: (Use arrow keys)
> default (babel, eslint)
  Manually select features

回车后,出现以下选择,在默认的基础上,再选择上 Router,Vuex,CSS,pre-processors

? Please pick a preset: Manually select features
? Check the features needed for your project:
 (*) Babel
 ( ) TypeScript
 ( ) Progressive Web App (PWA) Support
 (*) Router
 (*) Vuex
 (*) CSS Pre-processors
>(*) Linter / Formatter
 ( ) Unit Testing
 ( ) E2E Testing

回车后,将出现以下选项,这是问要不要启用history模式,可能会牵扯到项目的发布,一般在创建项目时，不会选择启用该 模式,根据情况在之后的项目中，再进行配置

? Please pick a preset: Manually select features
? Check the features needed for your project: Babel, Router, Vuex, CSS Pre-processors, Linter
? Use history mode for router? (Requires proper server setup for index fallback in production) (Y/n)

输入 n 回车,出现以下选择,这是选择使用哪种样式预编译语言,选择Less

? Pick a CSS pre-processor (PostCSS, Autoprefixer and CSS Modules are supported by default): (Use arrow keys)
> Sass/SCSS (with dart-sass)
  Sass/SCSS (with node-sass)
  Less
  Stylus

回车后,出现以下选择，这是与检查代码规范有关,选择第一个

? Pick a linter / formatter config:
> ESLint with error prevention only
  ESLint + Airbnb config
  ESLint + Standard config
  ESLint + Prettier

回车后,出现以下选择,这个与git有关,选择第一个

? Pick additional lint features: (Press <space> to select, <a> to toggle all, <i> to invert selection)
>(*) Lint on save
 ( ) Lint and fix on commit

回车后，出现以下选择，只是与配置有关,选择第一个

? Where do you prefer placing config for Babel, ESLint, etc.? (Use arrow keys)
> In dedicated config files
  In package.json

回车后,会出现以下选择，这是问是否保留本次选择,保留后在下一次创建项目时就可以直接使用本次选择

? Save this as a preset for future projects? (y/N)

输入 y 回车，会选择要保存到哪里,输入一个想要保存的位置,比如

? Save preset as:   ? Save preset as: D:\psh-workspace\vueconfitg

回车,然后等待项目创建完成

⚓  Running completion hooks...

📄  Generating README.md...

🎉  Successfully created project myvuedemo.
👉  Get started with the following commands:

 $ cd myvuedemo
 $ npm run serve

按照提示,执行命令

D:\psh-workspace>cd myvuedemo
D:\psh-workspace\myvuedemo>npm run serve

出现以下情况，说明vue项目搭建完成，并且运行起来了,之后对项目进行一些配置和修改即可。。。。。
