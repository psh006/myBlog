# BFC

# 概念:BFC全称‘Block Formatting Context’,中文意思：‘块级格式上下文’

# 理解：

BFC是一个独立的布局环境,可以理解为一个箱子(实际上看不见，摸不着）箱子内部的元素无论怎样都不会影响到外部。
转换为BFC的理解则是:BFC中的元素布局是不受外界的影响（我们往往利用这个特性来消除浮动元素对其非浮动元素
的兄弟元素和其子元素带来的影响，比如清除浮动）
并且在一个BFC中，块元素与行元素都会垂直的沿着其父元素的边框排列

# 触发 BFC

1.浮动元素(float) 除 'none' 以外的值
2.position的值不为' static'或者'relative'
3.display 不为 'none'
4.overflow 除了 'visible' 以外的值

# BFC的应用

1.解决浮动塌陷问题
2.自适应两栏布局
3.解决设置margin值重叠问题

# 总结

BFC就是页面上的一个隔离的独立容器，容器里面的子元素不会影响到外面的元素。它规定了元素与其他元素的关系和相互作用

# 浮动

# 什么是浮动

浮动是使用给节点添加float属性，最初的设计是用来实现文字环绕的

# 作用

添加了float的节点脱离文档流，同时触发节点的BFC，让节点往一个方向靠，并排成一行

# 副作用

当一个父节点的子节点全部浮动，就会造成父节点高度塌陷

# 清除浮动

1.给父节点主动添加高度值,
2.利用只要有一个子节点不浮动原理来添加一个不浮动的节点（通常使用伪元素before，after）.
3.触发父节点的bfc，常用的定位，或者overflow：hidden

# CSS里面相对单位(REM,EM,VW,VH）

REM：相对于根节点html的font-size
EM：父节点的font-size
VW：视口的宽度为 100VW，相对于把视口分为100份。
VH：视口高度为100VH，同理
以上单位都可以在移动端做页面适配，但通常使用REM和VW

# fixed是相对于谁定位的

fixed定位相对于浏览器视口来定位的

# 加上transform会出现的问题

添加上transform以后，fixed定位会失效。
如果fixed元素的祖先有transform属性,则fixed元素会相对与这个祖先计算,而不是视口（问题还在）

# style 内联元素 不推荐用style内联元素？内联元素有什么缺点？（css文件可以缓存）

style是节点的属性，不能被缓存；代码的可读性和可维护性相对弱一些，特别是多人协作开发的时候。
但是如果一个页面的style样式足够少的时候，可以使用style元素，因为一次请求最多携带14K的数据，如果足够小，还可以节约一次请求

# reflow(回流)与repain(重绘)

# 简述

reflow(回流):当元素的尺寸，DOM树结构发生改变时，浏览器会重新渲染页面
repain(重绘):当元素的样式(布局不发生color,opacity,visibility)发生改变的时候

# 触发回流

1.改变浏览器窗口的大小
2.元素尺寸，位置，内容发生改变
3.元素字体大小变化
4.添加或者删除可见DOM元素
5.激活CSS伪(如：hover)

# 避免 reflow(回流)：尽量减少DOM操作

# 重排一定会引起重绘，而重绘不一定会引起重排
